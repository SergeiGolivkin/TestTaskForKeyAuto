Необходимо реализовать микросервис управления учетными записями пользователей на основе поступающих извне данных о кадровых перемещениях.
Стек - spring boot (spring), SQL-СУБД для хранения данных (MySQL,MsSQL,PostgreSQL), брокер сообщений (Kafka, RabbitMQ, ActiveMQ) - опционально, будет плюсом, но не обязательно

Функция микросервиса - принять на входе JSON-объект с данными о кадровом перемещении, создать/изменить учетную запись, на основании полученных данных в JSON-объекте  и затем оттдать на выход JSON-объект с данными об учетной записи, которая была создана/модифицированна.

Примерная структура входного JSON-объекта:

{
	"id": NULL - для type = 1, INT - для type = 2,3
	"type": 1/2/3,
	"role" : "роль",
	"fio" : "ФИО",
	"post" : "Должность"
}

Примерная структура выходного JSON-объекта:

{
	"userId": id,
	"userName" : userName,
	"pass" : password
}

Система различает 3 типа кадровых перемещений, которые передаются в входном JSON-объекте в свойстве type:
1. Прием на работу - микросервис создает новую учетную запись для пользователя, на основании данных в JSON-объекте
2. Увольнение - микросервис блокирует учетную запись уволенного
3. Изменение / перевод - микросервис обновляет данные в учетной записи на основании данных в JSON-объекте

Данные о созданной/измененной учетной записи сохраняются в СУБД. После успешного сохранения в СУБД данные отдаются в виде выходного JSON-объекта.
Для случая реализации задания с использованием брокера сообщений - входной JSON-объект получать из подписки на топик в очереди, выходной JSON-объект публиковать в виде события в другой топик.
